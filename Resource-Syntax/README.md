## 🔬 Lab: Resource Syntax and Behavior

**Goal:** Understand how Terraform resource blocks are written, how implicit dependencies are formed through references, and observe all four resource behaviors.

---

### `c1-vesions.tf` — Terraform Block + Provider Block

```hcl
# Terraform Block
terraform {
  required_version = ">= 1.0.0"
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = ">= 2.0"
    }
  }
}

# Provider Block
provider "azurerm" {
  features {}
}
```

**Key learnings:**
- `terraform {}` block configures Terraform itself — required version and provider sources
- `required_version = ">= 1.0.0"` means any Terraform 1.x or above will work
- `provider "azurerm" {}` configures the Azure plugin — auth, defaults
- `features {}` is **mandatory** for the azurerm provider even when empty

---

### `c2-resource-group.tf` — Azure Resource Group

```hcl
# Resource-1: Azure Resource Group
resource "azurerm_resource_group" "myrg" {
  name     = "myrg-1"
  location = "East US"
}
```

**Key learnings:**
- Every `resource` block has **two labels**: type (`azurerm_resource_group`) + local name (`myrg`)
- Referenced by other resources as `azurerm_resource_group.myrg.name` / `.location` / `.id`
- This is the **root resource** — everything else depends on it

---

### `c3-virtual-network.tf` — VNet, Subnet, Public IP, NIC

```hcl
# Resource-2: Create Virtual Network
resource "azurerm_virtual_network" "myvnet" {
  name                = "myvnet-1"
  address_space       = ["10.0.0.0/16"]
  location            = azurerm_resource_group.myrg.location    # implicit dep → myrg
  resource_group_name = azurerm_resource_group.myrg.name        # implicit dep → myrg
  tags = {
    "Name" = "myvnet-1"
  }
}

# Resource-3: Create Subnet
resource "azurerm_subnet" "mysubnet" {
  name                 = "mysubnet-1"
  resource_group_name  = azurerm_resource_group.myrg.name       # implicit dep → myrg
  virtual_network_name = azurerm_virtual_network.myvnet.name    # implicit dep → myvnet
  address_prefixes     = ["10.0.2.0/24"]
}

# Resource-4: Create Public IP Address
resource "azurerm_public_ip" "mypublicip" {
  name                = "mypublicip-1"
  resource_group_name = azurerm_resource_group.myrg.name
  location            = azurerm_resource_group.myrg.location
  allocation_method   = "Static"
  tags = {
    environment = "Dev"
  }
}

# Resource-5: Create Network Interface
resource "azurerm_network_interface" "myvmnic" {
  name                = "vmnic-1"
  location            = azurerm_resource_group.myrg.location
  resource_group_name = azurerm_resource_group.myrg.name

  ip_configuration {                                            # nested block
    name                          = "internal"
    subnet_id                     = azurerm_subnet.mysubnet.id
    private_ip_address_allocation = "Dynamic"
    public_ip_address_id          = azurerm_public_ip.mypublicip.id
  }
}
```

---

### Dependency Chain (All Implicit)

```
azurerm_resource_group.myrg
         │
         ├──► azurerm_virtual_network.myvnet
         │              │
         │              └──► azurerm_subnet.mysubnet
         │                            │
         ├──► azurerm_public_ip.mypublicip   │
         │                                   │
         └──────► azurerm_network_interface.myvmnic ◄──(subnet.id + publicip.id)
```

Terraform reads these references, builds the dependency graph automatically, and provisions resources **in the correct order** — no `depends_on` needed here.

---

### Arguments vs Nested Blocks

| Type | Example | Description |
|------|---------|-------------|
| Argument | `name = "myvnet-1"` | Simple key = value |
| Argument (map) | `tags = { "Name" = "myvnet-1" }` | Key = map value |
| Nested Block | `ip_configuration { ... }` | Block inside a block — has its own schema |

---

### Files Generated by Terraform

| File | When Created | Purpose |
|------|-------------|---------|
| `.terraform/` | `terraform init` | Provider binaries downloaded from registry |
| `.terraform.lock.hcl` | `terraform init` | Pins exact provider version + checksums |
| `terraform.tfstate` | `terraform apply` | Records actual Azure resource IDs and state |
| `terraform.tfstate.backup` | Each `apply` / `destroy` | Previous state — automatic safety backup |
